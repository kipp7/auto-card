# 后台管理系统文档生成器

## 任务目标
根据项目的C端（用户端）功能，反推并生成后台管理系统的三个规划文档。

## 执行步骤

### 1. 扫描C端功能
- 查找所有前端页面、路由、API接口
- 识别业务模块：登录/注册 → 用户管理、订单 → 订单管理、商品 → 商品管理

### 2. 扫描数据模型
- 读取Model/Entity代码，提取表名、字段、类型、约束、索引

### 3. 扫描现有接口
- 记录C端接口的路径、参数、响应格式

## 生成规范

### 文档1：后台管理系统开发规划.md

格式：一级菜单：[模块名]
二级菜单：[页面名]
搜索字段：
字段名（输入框/下拉选择/日期范围）
列表字段：
字段1、字段2、字段3...
列表操作：
查看详情、编辑、删除、状态切换
新增/编辑表单字段：
字段名（必填/选填、类型、验证规则）
### 文档2：数据表设计.md
生成完整的MySQL 5.7+建表语句：
```sql
CREATE TABLE `表名` (
  `id` int(11) unsigned NOT NULL AUTO_INCREMENT COMMENT '主键',
  `字段名` 类型 NOT NULL DEFAULT '默认值' COMMENT '说明',
  PRIMARY KEY (`id`),
  KEY `idx_字段` (`字段`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='表说明';
```
### 文档3：API设计.md
包含两部分：
- **C端接口**：从实际代码提取
- **B端接口**：根据开发规划生成完整CRUD（列表、详情、新增、编辑、删除、状态修改）
## 核心要求
1. **从C端反推B端**：有什么C端功能就生成对应的后台管理模块
2. **三者一致**：开发规划、数据表、API接口的字段必须完全对应
3. **字段具体**：不使用xxx占位符，所有字段名明确
4. **可执行SQL**：建表语句必须符合MySQL 5.7+语法
5. **完整CRUD**：每个模块至少6个接口（列表/详情/新增/编辑/删除/状态）
## 输出要求
1. 先输出三个文档的完整内容
2. 立即创建docs目录并写入文件
3. 不等待用户确认，直接执行
